## Описание проекта

Цель проекта — разработать продвинутую систему прогнозирования итоговой стоимости страхового возмещения (**UltimateIncurredClaimCost**) с использованием современных методов машинного обучения и инструментов MLOps.

Датасет: **Workers Compensation** (OpenML ID: 42876) — 100 000 страховых дел с информацией о возрасте, зарплате, занятости, типе травмы и датах.

**Тип задачи:** регрессия; целевая переменная — стоимость выплаты в USD (диапазон: 20–3 100 000 долл., сильно правоскошенное распределение).

В рамках проекта реализованы:
- Углублённый анализ данных с временными трендами и проверкой нормальности
- Продвинутый Feature Engineering (9 новых признаков из дат, взаимодействий и бинарных флагов)
- Сегментация данных через K-Means, DBSCAN и доменные правила
- Сравнение 8 моделей: от линейной регрессии до ансамблевых (Stacking, Voting)
- Оптимизация гиперпараметров через Optuna (Байесовский поиск)
- K-Fold кросс-валидация и Early Stopping
- Интеграция с ClearML для управления экспериментами, версионирования данных и деплоя модели
- Интерпретируемость предсказаний: SHAP, LIME, Partial Dependence Plots
- Детальный анализ ошибок по ценовым сегментам

## Выполненные задачи

### Задача 1 — Сегментация данных и анализ по группам

**Описание:** разделение данных на сегменты и обучение специализированных моделей для каждого сегмента.

**Реализовано:**
- **K-Means кластеризация** (2–6 кластеров): метод локтя (WCSS), Silhouette Score = 0.118
- **DBSCAN** с PCA-предобработкой: автоматическое определение числа кластеров, выявление выбросов (36% шума)
- **Доменная сегментация** по начальной оценке: лёгкие (<$1K), средние ($1K–$10K), тяжёлые (>$10K)
- Сравнение общей модели и специализированных моделей по сегментам
- Визуализация через Plotly: boxplot, scatter, bar-chart метрик

---

### Задача 2 — Детальный анализ исходного датасета

**Описание:** создана отдельная страница Streamlit с углублённой аналитикой.

**Реализовано:**
- **Временной анализ:** тренды средней и медианной выплаты по годам (1985–2015), сезонность по месяцам
- **Распределение целевой переменной:** гистограммы (оригинал + log1p), тест нормальности D'Agostino–Pearson
- **Выбросы:** метод IQR — выявлено 16 458 выбросов (16.5% данных)
- **Корреляционный анализ:** heatmap (seaborn), топ-15 признаков по корреляции с целевой
- **Анализ категориальных признаков:** интерактивные boxplot по полу, семейному положению
- **Анализ начальных оценок:** scatter InitialCaseEstimate vs Cost (log-log), % попаданий в 2x-диапазон

---

### Задача 3 — Продвинутая предобработка и Feature Engineering

**Описание:** улучшение качества входных данных и создание новых информативных признаков.

**Реализовано:**

| Признак | Описание |
|---------|----------|
| `Age_x_WeeklyPay` | Взаимодействие: возраст × зарплата |
| `Estimate_per_Pay` | Отношение: начальная оценка / зарплата |
| `HasDependents` | Бинарный флаг: есть иждивенцы |
| `IsFullTime` | Бинарный флаг: полная занятость (≥35 ч/нед) |
| `Log_InitialEstimate` | Log1p-преобразование начальной оценки |
| `Accident_Year/Month/DayOfWeek` | Временны́е признаки из даты несчастного случая |
| `Reported_Year/Month/DayOfWeek` | Временны́е признаки из даты подачи |
| `ReportDelay_Days` | Задержка подачи заявки (дней) |

Итог: **13 исходных признаков → 22 признака** после FE.

- **Обработка выбросов:** IQR-метод + log1p-преобразование целевой переменной
- **PCA:** объяснённая дисперсия по компонентам, scatter PC1 vs PC2
- **t-SNE:** нелинейная визуализация 22 признаков в 2D на 1 000 наблюдениях

### Задача 4 — Выбор и обучение мощных моделей

**Описание:** сравнение базовых и ансамблевых моделей на одной тестовой выборке.

**Реализовано — базовые модели:**

| Модель | R² | MAE (USD) |
|--------|----|-----------|
| Linear Regression | 0.391 | $11 203 |
| Random Forest | 0.407 | $10 674 |
| Gradient Boosting | **0.441** | **$10 544** |
| XGBoost | 0.422 | $10 746 |
| LightGBM | 0.439 | $10 567 |
| CatBoost | 0.432 | $10 635 |

**Реализовано — ансамблевые методы:**

| Модель | R² | MAE (USD) |
|--------|----|-----------|
| Voting Regressor (XGB + LGB + RF) | 0.432 | $10 602 |
| Stacking (XGB + LGB + CAT → Ridge) | 0.440 | $10 567 |

Все модели сохраняются как `.pkl` в папку `models/`. Визуализация: grouped bar-chart метрик, scatter Факт vs Прогноз (log-log).

### Задача 5 — Оптимизация обучения модели

**Описание:** улучшение качества модели через кросс-валидацию, байесовскую оптимизацию и ранннюю остановку.

**Реализовано:**

**K-Fold Cross-Validation:**
- KFold с перемешиванием, пофолдовая визуализация R², линия среднего

**Optuna — байесовская оптимизация (TPE):**
- Оптимизация XGBoost и LightGBM (10–100 попыток)
- Пространство поиска: `n_estimators`, `max_depth`, `learning_rate`, `subsample`, `colsample_bytree`
- Лучший результат: R² = 0.444 (n_estimators=223, max_depth=3, lr=0.114)
- Визуализация: история оптимизации, важность гиперпараметров (FAnova)

**Early Stopping (LightGBM):**
- Максимум 500 деревьев, валидационная кривая по L2-потере
- Лучший результат: остановка на 178 итерации (сэкономлено 322 из 500), R² = 0.439

**Регуляризация L1/L2:** через параметры `reg_alpha` и `reg_lambda` в XGBoost (включены в пространство поиска Optuna).

### Задача 7 — Интерпретируемость модели

**Описание:** объяснение предсказаний модели на глобальном и локальном уровне.

**SHAP (SHapley Additive exPlanations):**
- Summary Plot (bar): топ-признаки по суммарному вкладу
- Beeswarm Plot: детальное распределение SHAP-значений по всей выборке
- Waterfall Plot: пошаговое объяснение отдельного предсказания
- Таблица SHAP-значений с сортировкой по модулю вклада

**LIME (Local Interpretable Model-agnostic Explanations):**
- Локальная линейная аппроксимация в окрестности выбранного наблюдения
- Интерактивный horizontal bar chart: зелёный = увеличивает предсказание, красный = уменьшает
- Сравнение прогноза и факта для выбранного случая

**Partial Dependence Plots (PDP):**
- 1D PDP: как изменяется средняя выплата при изменении одного признака (50 точек)
- 2D PDP: тепловая карта совместного влияния двух признаков (20×20)

---

### Задача 8 — Детальный анализ ошибок модели

**Описание:** выявление закономерностей в случаях, где модель ошибается.

**Реализовано:**
- Глобальные метрики на тестовой выборке: R²=0.422, MAE=$10 746, медиана ошибки
- **Анализ по 5 ценовым сегментам:** <$500, $500–$2K, $2K–$10K, $10K–$50K, >$50K
  - MAE (бар-чарт): наибольшая абсолютная ошибка в сегменте >$50K
  - MAPE (бар-чарт): наибольшая относительная ошибка в сегменте <$500 (малые выплаты непредсказуемы)
- **Паттерны в ошибочных случаях:** сравнение топ-500 ошибок с остальными по ключевым признакам
- **Интерактивный фильтр:** slider по порогу ошибки, scatter с подсветкой ошибочных случаев
